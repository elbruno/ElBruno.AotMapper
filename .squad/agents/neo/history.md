### Source Generator Implementation â€” Complete âś“

**What was implemented:**
- Created 5 core attributes in `ElBruno.AotMapper`:
  - `MapFromAttribute`: Marks destination type with source type and optional `Strict` mode
  - `MapToAttribute`: Marks source type to generate mapping TO destination
  - `MapPropertyAttribute`: Property-level renaming via `[MapProperty("SourceProp", "DestProp")]`
  - `MapIgnoreAttribute`: Marks properties to skip during mapping
  - `MapConverterAttribute`: Custom converter for specific property conversions
  - `IMapConverter<TSource, TDest>`: Interface for custom converters

- Created full incremental source generator in `ElBruno.AotMapper.Generator`:
  - **Models**: `MappingModel` and `PropertyMapping` with `MappingStrategy` enum
  - **DiagnosticDescriptors**: 5 analyzer rules (AOTMAP001-AOTMAP005)
  - **MappingParser**: Extracts mapping info from semantic model, handles property matching, type analysis, collection detection, enum detection, nested mapping detection
  - **MappingEmitter**: Generates extension methods with null checks, constructor mapping for records, object initializers for classes, collection mapping methods
  - **AotMapperGenerator**: Main entry point using `ForAttributeWithMetadataName` for both `[MapFrom]` and `[MapTo]`

**Technical choices:**
- Used `IIncrementalGenerator` with `ForAttributeWithMetadataName` for performance
- Generator targets `netstandard2.0` with `LangVersion: latest`
- Core library targets `netstandard2.0;net8.0;net10.0`
- All generated code is AOT-safe (no reflection, no `Activator.CreateInstance`, no `Expression.Compile`)
- Generated classes are `partial` so users can extend them
- Each generated file has `#nullable enable` and `<auto-generated/>` header
