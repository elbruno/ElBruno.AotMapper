# v0.6 — Generator Hardening, EF Projection, AOT Verified

We're thrilled to announce **ElBruno.AotMapper v0.6**, focused on hardening the source generator, shipping EF Core projection support, and verifying NativeAOT safety end-to-end.

## What's New

### Advanced Mapping Control

`[MapIgnore]` now works as expected—skip properties that don't belong in your DTO:

```csharp
[MapFrom(typeof(Customer))]
public record CustomerDto
{
    public string Id { get; init; }
    public string Name { get; init; }
    
    [MapIgnore]
    public string InternalNotes { get; init; }  // Excluded from mapping
}
```

`[MapConverter]` brings custom type conversion to the table. Plug in your own logic:

```csharp
[MapFrom(typeof(Order))]
public record OrderDto
{
    public string Id { get; init; }
    
    [MapConverter(typeof(DateTimeToStringConverter))]
    public string CreatedAtFormatted { get; init; }
}

public class DateTimeToStringConverter : IMapConverter<DateTime, string>
{
    public string Convert(DateTime source) => source.ToString("yyyy-MM-dd");
}
```

### EF Core Projections

Query your database and map in a single expression. When your projection translates cleanly to SQL, use the generated `ProjectToXxx()` extension:

```csharp
// One query, one Select—no N+1, no in-memory mapping
var orders = await context.Orders
    .Where(o => o.Status == OrderStatus.Completed)
    .ProjectToOrderDto()
    .ToListAsync();
```

The generator validates that your mapping patterns work with SQL, and emits a diagnostic if they don't.

### Generator Hardening

Fixed a critical performance issue: the incremental generator was capturing the entire `Compilation` object, throttling IDE responsiveness during editing. With this fix, incremental builds are snappy again.

We've also added **AOTMAP006** (inaccessible member) and **AOTMAP007** (EF projection not translatable) diagnostics, with HelpLinkUri on all rules for one-click access to remediation docs.

### Security First

Dependabot scans dependencies on every push. CodeQL runs security analysis on all code. GitHub Actions are pinned to specific commit SHAs to prevent supply-chain attacks. And NativeAOT publish is verified in CI, so every release is trim-safe and AOT-ready.

## Under the Hood

- Incremental generator caching now value-equals correctly for better hit rates
- `AddAotMapper()` has a new overload for future extensibility
- Dictionary<K,V> mapping now supported
- Enum↔int conversion added
